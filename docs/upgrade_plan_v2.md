# 可视化 AI 策略实验室升级计划书

**文档版本：** v2.0  
**作者：** Manus AI  
**日期：** 2026-03-20  

---

## 1. 升级背景与目标

当前的 Lighter DEX 交易系统采用硬编码的单实例架构，用户无法灵活调整 AI 的分析周期、数据源和策略逻辑。为了将系统从一个"固定的交易机器人"升级为**"灵活的 AI 策略平台"**，本次升级旨在实现：

1. **前端可视化创建 Agent**：用户可以在 Dashboard 上直接新建、配置、测试和发布多个独立的 AI 交易 Agent。
2. **多时间框架与多数据源**：支持为每个 Agent 独立配置 K 线周期（1m/5m/15m/1h/4h）和技术指标。
3. **策略沙盒测试**：在真实运行前，提供可视化的测试环境，展示 AI 的完整思考过程和决策输出。

---

## 2. 核心功能设计

### 2.1 策略实验室（Strategy Lab）页面

这是本次升级的核心前端页面，参考 VERGEx 模式设计，分为三大区域：

**左侧：参数与数据配置区**
- **基本设置**：Agent 名称、决策间隔（时间周期）。
- **风控参数**：仓位模式（全仓/逐仓）、最大杠杆、硬止损比例。
- **市场数据**：选择 K 线数据周期（1m, 5m, 15m, 1h, 4h, 1d）。
- **技术指标**：可勾选添加 RSI、MACD、Bollinger Bands 等指标，这些数据将随 K 线一起喂给 AI。

**中上：提示词逻辑编辑器（Prompt Editor）**
- 提供一个代码编辑器风格的文本框，用户可以直接编写 AI 的交易逻辑、止盈止损规则和信号过滤条件。
- 支持变量插值（如 `{{RSI}}`, `{{MACD}}`）。

**中下：测试与调试沙盒（Sandbox）**
- **测试控制**：选择测试模型、交易所数据源，点击「运行测试」。
- **三栏输出展示**：
  1. **提示词（输入）**：展示最终拼接好喂给 AI 的完整 Prompt（包含实时 K 线和指标数据）。
  2. **AI 思考（输出）**：展示 AI 的推理过程（Chain of Thought）。
  3. **AI 决策**：展示最终提取的结构化交易信号（方向、价格、止损止盈）。

### 2.2 Agent 管理中心

- **Agent 列表**：展示所有已创建的 Agent，包括草稿、运行中、已停止状态。
- **独立账户绑定**：每个发布的 Agent 必须绑定一个独立的 Lighter DEX 账户私钥，实现资金和仓位的物理隔离。
- **启停控制**：一键启动或暂停特定 Agent。

### 2.3 AI 决策看板（Decision Board）

- 集中展示所有运行中 Agent 的历史决策记录。
- 包含：时间、资产、周期、AI 思考摘要、最终动作（开仓/平仓/拒绝）。

## 3. 系统架构重构方案

为了支持前端动态创建 Agent，后端架构需要进行深度重构：

### 3.1 动态执行引擎（Dynamic Execution Engine）
- 废弃现有的单例 `Orchestrator` 主循环。
- 引入**任务调度器（Scheduler）**（如 APScheduler 或 Celery），根据数据库中配置的 Agent 列表，动态生成定时任务。
- 比如：Agent A 配置为 5 分钟周期，调度器每 5 分钟为其拉起一次分析流程；Agent B 配置为 1 小时周期，每小时拉起一次。

### 3.2 数据获取与指标计算层
- 接入 Binance API 获取全资产（含 XAUUSDT, XAGUSDT）的 K 线数据。
- 引入 `pandas-ta` 或 `TA-Lib` 库，根据前端配置动态计算 RSI、MACD 等技术指标。
- 将 K 线和指标数据格式化为 Markdown 表格，注入到用户自定义的 Prompt 中。

### 3.3 数据库模型扩展
新增以下核心表结构：
- `agents`：存储 Agent 基本信息、状态、私钥。
- `agent_configs`：存储时间周期、风控参数、技术指标选择。
- `agent_prompts`：存储用户编写的提示词逻辑。
- `ai_decisions`：存储每次执行的完整输入输出日志。

## 4. 实施步骤与里程碑

本项目建议分三个阶段实施：

### 阶段一：底层引擎与数据源重构（预计 3 天）
1. 接入 Binance K 线数据和技术指标计算库。
2. 重构后端架构，实现基于数据库配置的动态任务调度（支持多实例并行）。
3. 改造 `position_monitor`，支持多账户隔离持久化。
*里程碑：后端具备同时运行多个不同周期、不同策略 Agent 的能力。*

### 阶段二：策略实验室前端开发（预计 3 天）
1. 开发「策略实验室」页面，实现参数配置、Prompt 编辑器。
2. 开发「沙盒测试」API，实现前端点击测试后，后端拉取实时数据并调用 LLM，返回三栏结果。
*里程碑：用户可以在前端编写策略并实时测试 AI 的反应。*

### 阶段三：Agent 管理与决策看板（预计 2 天）
1. 开发 Agent 列表页，实现创建、发布、启停功能。
2. 开发 AI 决策看板，展示历史决策记录。
3. 联调测试，确保前端配置能正确下发到执行引擎并稳定运行。
*里程碑：全新可视化 AI 策略平台正式上线。*

---

## 5. 优势与价值

1. **极高的灵活性**：无需修改代码即可尝试各种交易策略和时间周期。
2. **所见即所得**：沙盒测试功能让 AI 的行为变得可预测、可调试。
3. **策略隔离**：多账户机制完美解决了不同周期策略之间的仓位冲突问题。
